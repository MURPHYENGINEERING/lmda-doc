With the special case of a function that\textquotesingle{}s declared in one module (the {\itshape interface} module) and defined in another (the {\itshape implementation} module)\+: {\ttfamily lmda\+\_\+interface\+\_\+function\+\_\+impl()}. Here the prefix {\ttfamily lmda} indicates the broadest scope, and the suffix {\ttfamily impl} indicates the separated declaration and definition.\hypertarget{md_readme_autotoc_md9}{}\doxysubsection{Application Design}\label{md_readme_autotoc_md9}
At the first level, the game uses a {\bfseries{plugin}} architecture. The {\bfseries{application}} hosts two plugins\+: the {\bfseries{game plugin}} simulates the game world, and the {\bfseries{presentation plugin}} represents the game world to the outside world.

There are two presentation plugins\+: the {\bfseries{SDL plugin}} implements a 2D game, and the {\bfseries{server plugin}} implements a headless server with a browser-\/accessible dashboard. The server is headless because it still runs the game plugin, meaning it still constitutes a game client playing the game; but the only inputs and outputs are from and to the network and AI.\hypertarget{md_readme_autotoc_md10}{}\doxysubsection{Project Structure}\label{md_readme_autotoc_md10}
